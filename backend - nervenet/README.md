# Nervenet: AI Electricity Analytics & Customer Support System

Nervenet is a production-ready AI customer support, database management, and data analytics system built for the electricity distribution meter department. It features a conversational interface that interacts with an existing MySQL database through a custom Model Context Protocol (MCP) server, secured by an automated, stateful **Privacy Engine**.

---

## Technical Stack

* **Backend Engine**: Django 6.0.6 (orchestrates conversation sessions, prompt engineering, history tracking, and security mappings).
* **Database Layer**: Existing production MySQL database (`analytics_demo` table structure).
* **AI Protocol Gateway**: custom Model Context Protocol (MCP) Server built with FastMCP.
* **LLM Client Integration**: Anthropic Claude API (Claude-3.5-Sonnet / Claude-Opus).
* **User Interface**: Streamlit web chat application.
* **Security & Anonymization**: In-memory and session-persisted Privacy Engine (regular-expression and key-based tokenization).

## Quick Start Guide

### 1. Setup Environment
Clone the repository and activate your virtual environment:
```bash
# Activate your virtual environment
venv\Scripts\activate
```

Install requirements (if not already installed):
```bash
pip install -r requirements.txt
```

### 2. Configure Credentials
Create/edit the `.env` file at the root of the project directory:
```env
# Anthropic API Key
ANTHROPIC_API_KEY=your-api-key-here

# Production MySQL Database Connection details
DB_HOST=13.235.168.247
DB_PORT=3306
DB_USER=remote_root
DB_PASSWORD=your-mysql-password
DB_NAME=analytics_demo
```

### 3. Run Automated Tests
Verify that all unit and integration tests (including Django conversation engine, PrivacyEngine, and MCP registry) pass:
```bash
python -m pytest
```

### 4. Launch the Applications
Run the Django backend server (listens on `http://127.0.0.1:8000/`):
```bash
python manage.py runserver
```

Run the Streamlit frontend client (opens `http://localhost:8501` in your browser):
```bash
streamlit run chat_app.py
```

## Secure Data lifecycle

To ensure strict compliance with user data privacy regulations, all PII (Personally Identifiable Information) is automatically masked before it is sent to the LLM:
* **Ingress Prompt Masking**: User prompts containing UIDs or 10-digit phone numbers are replaced with tokens (e.g. `<//UID-abc123xyz456//>`).
* **Tool Query Translation**: Database queries generated by Claude are decrypted in memory immediately before execution, allowing standard database searches.
* **Query Result Encryption**: Row columns containing names, mobile numbers, addresses, or GPS coordinates are encrypted to tokens before Claude receives them.
* **Egress Decryption**: Claude writes its final text responses using tokens, which are decrypted by the Django backend just-in-time before rendering to the user interface.
